# Documentation updates{#documentation-updates}

## Last updated: March 7, 2019 {#March-2019-update}

* Questo aggiornamento si riferiva principalmente alla release 2.2 Media SDK sulle piattaforme javascript e OTT.
* La release 2.2 Media SDK su piattaforme javascript e OTT fornisce lo stesso supporto descritto di seguito per le piattaforme iOS e Android (aggiornamento del 1 novembre 2018).

## Last updated: November 1, 2018 {#November-2018-update}

* Questo aggiornamento si riferisce principalmente alla versione 2.2 di Media SDK sulle piattaforme Android e iOS.
* La versione 2.2 di Media SDK su Android e iOS fornisce il supporto per il tracciamento dell'audio su queste
piattaforme, insieme a miglioramenti interni.
* Con l'aggiunta del tracciamento audio e con le funzionalità
di tracciamento audio e video ora disponibili sia in Media SDK che nell'API Media Collection, viene richiamato un aggiornamento di denominazione relativamente grossualmente
per:

   * La soluzione globale presenta i titoli di Adobe Analytics per audio e video
   * La abbreviazione precedentemente utilizzata in tutti i documenti, "Video Analytics", è ora "Media Analytics"
   * Nell'SDK, i riferimenti a «Video Heartbeat Library (VHL)» sono ora «Media SDK»
   * Nomi file e URL (ad es., collegamenti alle API di riferimento) che precedentemente fanno riferimento a "video" o "vhl" in cui è stato utilizzato "Supporto"
   * Nel codice, i nomi dei tasti di metadati ora includono "File multimediali" invece di "VIDEO"
   * e così via…

* Along with the above, some additional restructuring has occurred in the Media SDK section, including Standard Metadata implementation and reference returning to their own topics (they had been absorbed into the *Track Core* topics in the previous doc update). These topics, along with the *Track core*, and *Track seeking*, and *Track buffering* topics are now grouped together under *Track audio and video playback*.

* Il modulo Federated Analytics è stato aggiornato alla Versione 3.2 per riflettere i nuovi parametri relativi all'audio di tracciamento.

## Update: October 10, 2018 {#October-2018-update}

* La struttura del documento è stata «refactorizzata» nell'area di implementazione dell'SDK, combinando le guide di implementazione singole (ma più identiche) in una sezione di implementazione dell'SDK, con esempi di tracciamento specifici della piattaforma rappresentati nelle sottosezioni sotto argomenti di tracciamento comuni.
* I file sono stati rinominati nell'attesa di una migrazione a un nuovo sistema doc. Tutti i prefissi DITA (c_, r_, t_) indicano rispettivamente i tipi di concetti, riferimenti e attività). Tutti i caratteri di sottolineatura ('_') sono stati sostituiti con trattini ('-'). Inoltre, i nomi dei file sono più simili ai titoli degli argomenti.
* Aggiornamenti agli argomenti generali di convalida e certificazione.
* Nuovo materiale introduttivo, inclusa una presentazione delle opzioni di misurazione, insieme agli aggiornamenti ai prerequisiti, ai percorsi di implementazione e all'abilitazione di Audience Manager.
* Aggiornamenti alle sezioni Metriche e Metadati e Reporting e analisi, che riflettono l'aggiunta delle funzionalità di Analytics audio.
